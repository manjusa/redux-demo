1. The Redux Store (src/app/store.js)

The store is the single source of truth for your app's state. Think of it as a global database that any component can access.

// Create the store with configureStore (Redux Toolkit) export const store = configureStore({ reducer: { handles: handlesReducer, // Each reducer manages a slice of state }, preloadedState, // Load saved data from localStorage });
Key Concepts:
  • Single Store: Your entire app has ONE store that holds ALL state
  • Reducers: Pure functions that specify how state changes in response to actions
  • preloadedState: Initialize store with data (like from localStorage)

    • reducer: This defines the slice reducers that will be combined to create the root reducer. It can be:
      • 1) A single Reducer Function: When you pass a single reducer function, that function becomes the entire root reducer for your store: 
        // A simple counter reducer function
function counterReducer(state = { count: 0 }, action) {
  switch (action.type) {
    case 'INCREMENT':
      return { count: state.count + 1 };
    case 'DECREMENT':
      return { count: state.count - 1 };
    default:
      return state;
  }
}

// Using single reducer
const store = configureStore({
  reducer: counterReducer // Single function
});

// Your state shape will be: { count: 0 }
      • 2) Object of Reducers (Slice Reducers) - An object where keys become state slice names and values are the reducer functions. When you pass an object, Redux automatically combines them using combineReducers.(Btw reducers exist in "slice.js"). Each key becomes a slice of your state:

2. Slices - Organizing State Logic (src/features/handles/handlesSlice.js)

A slice is a collection of Redux logic for a specific feature.

  • import from "@reduxjs/toolkit";
  • A slice is a Redux Toolkit concept that contains:
    Initial State - The starting value for this piece of state
    reducers - Functions that define how the state updates
    Action Creators - Automatically generated functions to dispatch actions

    • How Slice Links to Store 
  • 
            1) CREATE SLICE (handlesSlice.js)
            const handlesSlice = createSlice({
                name: 'handles', // Slice name
                initialState: [], // Initial state for this slice
                reducers: { // Reducer functions
                    addHandle: (state, action) => {
                        state.push(action.payload);
                    },
                    removeHandle: (state, action) => {
                        return state.filter(handle => handle.id !== action.payload);
                    }
                }
            });

            // 2. EXPORT REDUCER
            export default handlesSlice.reducer; // This goes to store
            export const { addHandle, removeHandle } = handlesSlice.actions; // Action creators

            // 3. CONNECT TO STORE (store.js the one we created above)
            import handlesReducer from './handles/handlesSlice';

            const store = configureStore({
                reducer: {
                    handles: handlesReducer // Links slice reducer to store
                }
            });

Am asking about syntax. reducer(state, action) { state.items.push(action.payload); }, while updateHandle directly takes state.action

The Syntax Difference:

addHandle (Object with reducer property)

:When you have BOTH reducer AND prepare: You MUST use the object syntax 

addHandle: {
                    reducer(state, action) {  // ‚Üê function inside an object
                        state.items.push(action.payload);
                    },
                    prepare({ platform, username }) {
                        // ...
                    }
                    }
updateHandle (Direct function):
updateHandle(state, action) {  // ‚Üê direct function
                const { id, platform, username } = action.payload;
                // ...
                }

When you have ONLY a reducer: like in update ,shorthand sytnax would do 

// This is shorthand syntax
updateHandle(state, action) {
  // just the reducer logic
}

// Redux Toolkit automatically converts this to:
updateHandle: {
  reducer(state, action) {
    // your logic here
  }
}

1. The Redux Store

The store is the single source of truth for your app's state. Think of it as a global database that any component can access.

Create store.js file
Create store.js file in the "app" folder: src/app/store.js
Configure the store variable
Create "store" from configureStore
Creates store from configureStore. configureStore takes several options; here we highlight the important ones:

reducer: This defines the slice reducers that will be combined to create the root reducer. It can be:
Single Reducer Function
When you pass a single reducer function, that function becomes the entire root reducer for your store.
Object of Reducers (Slice Reducers)
An object where keys become state slice names and values are the reducer functions. Redux automatically combines them using combineReducers. Each key becomes a slice of your state.
preloadedState (Optional)
preloadedState: (optional) - The initial state value to be used when the store is created. Useful to hydrate state from localStorage or server-rendered state.
create Local Storage Key, loadState() and then save it in preloadedState
a) Create a const LOCALSTORAGE_KEY and give it some value for storing/retrieving data.
b) loadState()- that loads data (serialized) from "localStorage" using above key. Use try catch etc.
c) basically store the state in preloadedState variable
Store Subscription
Call subscribe on store variable. The subscribe method runs the provided callback function every time the store's state changes.
Save Function
Create saveState(<>)
- takes curr "state" as parameter &
- saves to "localStorage" using above key. Use try catch etc. Then expose the store.

2. Slices - Organizing State Logic

A slice is a collection of Redux logic for a specific feature.

A slice is a Redux Toolkit concept that contains:
Initial State - The starting value for this piece of state
reducers - Functions that define how the state updates
Action Creators - Automatically generated functions to dispatch actions
Selectors-that returns data

                import "./index.css";

Reading State with useSelector (src/features/handles/HandleList.js)

import { useSelector } from "react-redux"; import { selectAllHandles } from "./handlesSlice"; export default function HandleList() { const handles = useSelector(selectAllHandles); // Gets state from store return ( <div> {handles.map(handle => <HandleItem key={handle.id} handle={handle} />)} </div> ); }

Dispatching Actions with useDispatch (src/features/handles/AddHandleForm.js)

import { useDispatch } from "react-redux"; import { addHandle } from "./handlesSlice"; export default function AddHandleForm() { const dispatch = useDispatch(); // Get dispatch function function onSubmit(e) { e.preventDefault(); dispatch(addHandle({ platform, username })); // Send action to store } return <form onSubmit={onSubmit}>...</form>; }

üîÑ Complete Redux Data Flow

Here's what happens when you add a new handle:

1. User Action: User submits form in AddHandleForm
2. Dispatch: dispatch(addHandle({ platform: "twitter", username: "@john" }))
3. Prepare: Prepare function adds ID, timestamp, normalizes username
4. Reducer: Reducer receives prepared payload and adds to state.handles.items
5. Store Update: Store state is updated with new handle
6. Auto-Save: store.subscribe() callback saves to localStorage
7. Component Re-render: HandleList re-renders because it uses useSelector(selectAllHandles)
8. UI Update: New handle appears in the list

üîë Key Redux Concepts Learned

  • Store: Single source of truth for app state
  • Actions: Objects describing what happened (with type and payload)
  • Reducers: Pure functions that calculate new state based on actions
  • Slices: Modern way to organize Redux logic with Redux Toolkit
  • Selectors: Functions to read specific pieces of state
  • useSelector: React hook to read state from store
  • useDispatch: React hook to send actions to store
  • Provider: React component that makes store available to all components
  • Immutability: Redux requires immutable updates (Redux Toolkit uses Immer under the hood)
